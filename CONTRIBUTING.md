# Contributing to SPAN API Documentation

Thank you for your interest in contributing to the SPAN API documentation!

## How to Contribute

### Reporting Issues

Use [Issues](https://github.com/spanio/SPAN-API-Client-Docs/issues) to report:

- Bugs in the SPAN API itself (unexpected behavior, errors, crashes)
- Errors or omissions in the documentation

Issues are not the place for questions, feature requests, or general discussion.

### Discussions

Use [Discussions](https://github.com/spanio/SPAN-API-Client-Docs/discussions) for:

- Questions about SPAN API usage
- Feature requests and suggestions
- Sharing integration ideas and projects
- Asking other SPAN API users for advice

Discussions are more free-form and open—a good place to connect with the developer community.

### Pull Requests

We welcome pull requests for:

- Documentation corrections or clarifications
- Improvements to example scripts
- Additional usage examples

**Note:** Pull requests are for contributions to this repository (documentation and scripts)—not for SPAN API feature requests or changes. For feature requests, please use [Discussions](https://github.com/spanio/SPAN-API-Client-Docs/discussions).

### Before Submitting a PR

1. Fork the repository
2. Create a feature branch from `main`
3. Make your changes
4. Test any code examples to ensure they work
5. Submit a pull request with a clear description of your changes

## Code of Conduct

Please be respectful and constructive in all interactions. We're here to help developers integrate with SPAN panels successfully.

## Important Notice

SPAN API is provided on an "as-is" basis. Updates and maintenance, including responses to issues filed on GitHub, will take place on an "as time and resources permit" basis.

**SPAN customer support does not provide assistance for API usage, troubleshooting, or integrations.**
